# 캠퍼스 라이프 RPG 게임 (Campus Life RPG)

## 🎮 프로젝트 소개

대학생활을 좀 더 재밌게 하고자 학교 생활 활동을 기반으로 스탯과 경험치를 쌓아 나의 캐릭터를 레벨업하는 "캠퍼스 라이프 RPG" 게임입니다.

**개발 기간:** 약 2달 (8주 로드맵)
**팀 구성:** 개발자 3명

## 🛠 기술 스택(추후 변동 예정)

- **Frontend:** React, HTML, CSS
- **Backend:** Node.js (Express)
- **Database:** MySQL
- **Tools:** DBeaver / VS Code, `uv` (가상환경)

## 📑 주요 페이지

1. **메인 페이지:** 닉네임, 코인, 상점/설정 아이콘, 내 캐릭터/레벨/칭호, 하단 메뉴
2. **퀘스트 페이지:** 일일/주간 퀘스트 (보상: 코인, 스탯 점수), 완료 시 취소선
3. **투두리스트 페이지:** 일정 추가 및 완료 관리 (보상: 경험치)
4. **스탯 페이지:** 오각형 레이더 차트, 스탯 점수 나열 및 올리기, 일일 제한 횟수
5. **친구창 페이지:** 친구 추가/삭제, 타 유저 정보(닉네임, 레벨, 칭호)
6. **상점 페이지:** 보유 코인, 2열 아이템 그리드 (예: 경험치 부스터)

## 📈 캐릭터 성장 시스템

- **경험치 (레벨):** 투두리스트 일정 수행 시 획득
- **스탯:** 퀘스트 수행 시 획득 (일일 제한 있음)
- **코인:** 퀘스트 수행 시 획득
- **진화:** 스탯을 일정 단계 올렸을 시 진화 (기본 대학생 -> ?)
- **레벨 제한:** 5레벨 단위로 올릴 수 있는 최대 스탯 상한선 지정

## 🚀 상세 프로젝트 계획 (8주 로드맵)

- **Phase 1 (1~2주 차):** 데이터베이스(DB) 설계 (유저, 스탯, 퀘스트/투두, 상점 테이블)
- **Phase 2 (3~4주 차):** 백엔드(API) 개발 (인증, 성장 로직, 퀘스트 시스템, 상점/친구 API)
- **Phase 3 (5~6주 차):** 프론트엔드(UI/UX) 구현 (React 시각화, Chart.js 도입)
- **Phase 4 (7~8주 차):** 통합 및 테스트, 예외 처리, 최종 배포 (AWS/Vercel)

## 💻 협업 및 개발 환경 구축 안내

본 프로젝트는 3명의 개발자가 함께 협업하는 프로젝트입니다. 아래 가이드에 따라 환경을 세팅해주세요.

### 1. 가상환경 설정 (`uv` 사용)

이 프로젝트는 패키지 관리 및 격리된 환경을 위해 `uv`를 사용합니다. 프로젝트를 처음 클론(Clone)받은 팀원들은 아래 순서대로 환경을 세팅해주세요.

**1) 가상환경 생성 및 활성화**

```bash
# 가상환경 생성
uv venv

# 가상환경 활성화 (Mac/Linux)
source .venv/bin/activate

# 가상환경 활성화 (Windows)
.venv\Scripts\activate
```

*(활성화 후 터미널 프롬프트 앞에 `(.venv)`가 표시되는지 확인하세요.)*

**2) 파이썬 패키지 설치**
가상환경이 켜진 상태에서 `requirements.txt`에 명시된 필수 패키지들을 설치합니다.

```bash
uv pip install -r requirements.txt
```

### 2. 폴더 구조

- `frontend/`: React 기반의 프론트엔드 코드가 위치합니다.
- `backend/`: Node.js(Express) 기반의 백엔드 코드가 위치합니다.

### 3. 협업 규칙 (Git & GitHub)

- **브랜치 전략:** `main` (배포용) -> `develop` (개발용) -> `feature/기능명` (각자 작업)
- **커밋 컨벤션:**
  - `feat:` 새로운 기능 추가
  - `fix:` 버그 수정
  - `docs:` 문서 수정
  - `style:` 코드 포맷팅, 세미콜론 누락 등 (코드 변경 없음)
  - `refactor:` 코드 리팩토링
- **PR (Pull Request):** 작업 완료 후 `develop` 브랜치로 PR을 생성하고, 팀원 1명 이상의 리뷰(Approve)를 받은 후 Merge 합니다.

### 4. 실행 방법 (예정)

node.js를 설치 후 진행해주세요

- Backend: `cd backend && npm install && npm start`
- Frontend: `cd frontend && npm install && npm run dev`
